01

總覽與定位

IP 位址是網路層的「門牌號」；子網路遮罩決定哪些位元是「街道」、哪些是「房號」。

上一篇我們建立了 TCP/IP 分層心智模型，知道網路層（Internet Layer）用 IP 位址回答「封包要送到哪一台主機」。但光有 32 位元的數字還不夠——你必須知道：

  • 這台主機跟我是同一個子網路嗎？（同網段可直接 ARP，跨網段要丟給 Gateway）
  • 這個子網路從哪到哪？（網路位址、廣播位址、可用主機範圍）
  • 機房新上架一台伺服器，IP、遮罩、Gateway 該填什麼？

本篇聚焦 IPv4 定址與子網路劃分（Subnetting）——這是後續「區域網路與設備」「NAT／Port Forwarding」的先備地基。貫穿情境：新機房上架一台伺服器，你要手動設定固定 IP、子網路遮罩與預設閘道。

座標這篇在系列中的位置

承接「TCP/IP 分層模型」的網路層（L3）細節。上一篇說 IP 負責跨網路定址與路由；本篇回答「IP 位址本身怎麼解讀、子網路遮罩到底在算什麼」。與「區域網路與設備」的差異化：本篇講邏輯定址與子網路劃分，不深入 Switch／MAC／ARP 的實體接線。

一句話答法IPv4 是 32 位元門牌號；子網路遮罩把位址切成「網路前綴＋主機號」，決定同網段判斷、可用主機範圍與 Gateway 該指向哪；CIDR 用斜線前綴簡寫遮罩長度。
02

IPv4 位址結構

32 位元、四組八位元、點分十進位——看懂這三個數字就能讀懂任何 IP。

IPv4 位址是 32 位元（bit）的整數，慣例切成四段、每段 8 bit，用點分十進位（dotted decimal）書寫，例如 192.168.1.10。每一段範圍 0–255，因為 28 = 256。

在還沒劃分子網路之前，早期 IP 用類別定址（Classful）：A 類前 8 bit 是網路、B 類前 16 bit、C 類前 24 bit。但這太僵化——一個 C 類網段固定 254 台主機，小公司浪費、大公司不夠。現代網路全面改用無類別定址（Classless），靠子網路遮罩長度彈性切分，也就是 CIDR。

表示法範例說明
點分十進位192.168.1.10人類可讀，設定檔最常見
二進位11000000.10101000.00000001.00001010子網路運算的本質形式
32 位元整數3232235786程式內部常用 inet_aton 轉換
CIDR 前綴192.168.1.10/24IP ＋ 遮罩長度，路由表標準寫法

面試常考：把 10.0.0.0/8 翻成二進位，前 8 bit 是網路、後 24 bit 是主機——可用主機數 224 − 2（扣網路位址與廣播位址）。

陷阱特殊位址別拿來配給主機

0.0.0.0（本機未指定）、127.0.0.0/8（loopback）、255.255.255.255（有限廣播）、以及每個子網路的網路位址（全 0 主機段）與廣播位址（全 1 主機段）都不可指派給一般主機。

一句話答法IPv4 是 32 bit，切成四段八位元用點分十進位表示；現代用無類別 CIDR 取代僵化的 A/B/C 類，彈性決定網路與主機各佔幾 bit。
03

子網路遮罩：到底在算什麼

遮罩是一面「篩子」——跟 IP 做 AND，網路位元留下、主機位元歸零，就得到網路位址。

子網路遮罩（Subnet Mask）跟 IP 位址等長（32 bit），由連續的 1（網路部分）與連續的 0（主機部分）組成。核心運算只有一個：網路位址 = IP AND Mask。

以 192.168.1.10 搭配遮罩 255.255.255.0（即 /24，前 24 bit 為 1）為例：

Network = IP & Mask → 192.168.1.0

遮罩告訴你三件實務上最重要的事：

  • 同網段判斷：兩台主機的 IP 分別 AND 同一遮罩，結果相同 → 同子網路，可直接 L2 通訊；不同 → 封包要交給 Gateway 轉發。
  • 可用主機範圍：主機段全 0 是網路位址、全 1 是廣播位址，中間才是可指派給主機的位址。
  • 子網路大小：主機 bit 數 h，可用主機數 = 2h − 2（IPv4 慣例扣掉網路與廣播）。
遮罩（點分）CIDR主機 bit 數可用主機數
255.255.255.0/248254
255.255.255.128/257126
255.255.255.192/26662
255.255.0.0/161665534

機房情境：若機房給你 10.20.30.0/26，代表整段只有 62 台可用主機——規劃 IP 時別超出範圍。

關鍵廣播位址怎麼算

廣播位址 = 網路位址 OR（遮罩的反碼）。以 192.168.1.0/24 為例，廣播是 192.168.1.255。往這個位址送的封包，子網路內所有主機都會收到。

一句話答法子網路遮罩用連續 1 標記網路前綴；IP AND Mask 得網路位址，決定同網段與路由；主機 bit 全 0／全 1 分別是網路位址與廣播位址，中間才可配給主機。
04

CIDR 表示法

斜線後的數字就是「前面幾個 bit 是網路」——路由表、防火牆、雲端 VPC 都這樣寫。

CIDR（Classless Inter-Domain Routing）把 IP 與遮罩長度寫在一起：10.20.30.0/26 表示前 26 bit 是網路前綴、後 6 bit 是主機。好處是簡潔、無歧義，路由聚合（route summarization）也靠它把多條小網段合成一條。

常見對照：

  • /32 → 單一主機（host route），遮罩 255.255.255.255
  • /24 → 典型區網，254 台主機
  • /16 → B 類等效，65534 台主機
  • /8 → A 類等效，16777214 台主機
陷阱遮罩必須是「左連續 1」

合法遮罩的二進位必須是 111…1100…00 的形式，例如 255.255.255.240（/28）合法，255.255.255.129 不合法——中間不能出現 0 夾在 1 之間。設定錯誤的遮罩會導致主機無法判斷同網段。

一句話答法CIDR 在 IP 後加斜線前綴長度（如 /24），一眼看出網路佔幾 bit；路由表與雲端安全群組規則普遍採用此寫法。
05

Gateway 與預設閘道

同網段直送、跨網段交給 Gateway——它是子網路通往外界的「出口路由器」。

當主機要送封包給另一個 IP，作業系統先做同網段判斷：

  1. 目的 IP AND 子網路遮罩，跟自己的 IP AND 遮罩比較。
  2. 相同 → 目的在同子網路，用 ARP 找對方 MAC，直接 L2 送達。
  3. 不同 → 封包交給預設閘道（Default Gateway），通常是子網路內某台路由器的介面 IP。

機房上架伺服器時，網管通常會給你三個值：

設定項範例角色
IP Address10.20.30.15這台主機在子網路內的門牌
Subnet Mask255.255.255.192（/26）界定同網段範圍
Default Gateway10.20.30.1子網路出口，通常是 .1 或 .254

Gateway 必須跟主機在同一子網路內——你把 Gateway 設成不同網段的 IP，主機永遠找不到它。

實務為什麼 Gateway 常是 .1？

純粹慣例，不是協定規定。很多機房把路由器介面設在子網路的第一個可用位址（如 x.x.x.1），方便記憶與文件化。重點是它必須落在你的可用主機範圍內，且確實有一台路由器在聽。

一句話答法預設閘道是子網路的出口路由器 IP；同網段直送、跨網段把封包交給 Gateway，Gateway 必須與主機同子網路且位在可用主機範圍內。
06

靜態 IP vs DHCP

伺服器、印表機、網路設備要固定；筆電、訪客 Wi-Fi 要自動——依角色取捨。

DHCP（Dynamic Host Configuration Protocol）讓主機開機時自動向 DHCP Server 索取 IP、遮罩、Gateway、DNS，省去手動設定。家用路由器、企業區網、雲端 VPC 的 DHCP 都是同一原理。

靜態 IP（Static）則是手動寫死三件套（IP／Mask／Gateway），或透過 DHCP Reservation 把特定 MAC 綁定固定 IP——效果一樣：位址不變。

面向靜態 IPDHCP
適用對象伺服器、DB、路由器、印表機、監控筆電、手機、訪客、一般辦公 PC
可預測性高——位址固定，防火牆／DNS 好管理租約到期可能換 IP（除非 Reservation）
維運成本需手動規劃、避免衝突集中管理，但需維護 DHCP Server
故障排查設定錯誤就是錯，不會自動修正DHCP 掛掉 → 新主機拿不到 IP
陷阱靜態 IP 衝突（IP Conflict）

兩台主機設了相同 IP，後開機的會偵測到衝突而無法正常通訊。機房規劃時應維護 IP 清冊，或把伺服器 MAC 登記到 DHCP Reservation，避免人工填錯。

一句話答法伺服器與基礎設施用靜態 IP 或 DHCP Reservation 確保位址固定；終端裝置用 DHCP 降低維運負擔；靜態設定要防 IP 衝突。

08

面試與常見誤區速查

高頻子網路題、易混清單、一句話收尾。

高頻問答（點擊展開）

Q1 /24 子網路有幾個可用 IP？▸

主機 bit = 32 − 24 = 8，理論 28 = 256 個位址，扣掉網路位址（.0）與廣播位址（.255），可用 254 個。面試記公式：2h − 2，h 是主機 bit 數。

Q2 怎麼判斷兩台主機是否同網段？▸

兩台 IP 分別與同一子網路遮罩做 AND，結果相同即同網段。例如 192.168.1.10/24 與 192.168.1.200/24 的網路位址都是 192.168.1.0，同網段；若一方是 /25 則可能不同。

Q3 Gateway 可以設成網路位址或廣播位址嗎？▸

不行。Gateway 必須是可用主機範圍內的一個合法單播位址。設成 x.x.x.0（網路位址）或 x.x.x.255（廣播位址）會導致無法正常轉發。

Q4 /30 和 /31 有什麼差別？為什麼點對點常用 /30？▸

/30 有 4 個位址、扣掉網路與廣播剩 2 個可用——剛好路由器兩端各一。傳統上 /31（RFC 3021）在點對點鏈路可省略網路與廣播，兩個位址都當主機用，但面試與實務題仍以 /30 最常見。

Q5 靜態 IP 和 DHCP Reservation 有什麼不同？▸

效果都是「這台機器永遠拿到同一個 IP」。靜態是在主機本機手動填；Reservation 是在 DHCP Server 把 MAC 綁定 IP，主機仍走 DHCP 流程但每次都拿到保留位址。伺服器推薦 Reservation 或靜態，方便集中管理又不失去可預測性。

易混與陷阱清單

坑以為遮罩可以任意填——必須是左連續 1，255.255.255.129 這類不合法。
坑忘記扣網路與廣播位址，說 /24 有 256 個可用 IP（正確是 254）。
坑Gateway 設在不同子網路——主機 ARP 不到，對外全斷。
坑把 CIDR 前綴與子網路遮罩搞混——/24 等價於 255.255.255.0，不是 255.255.0.0。
坑靜態 IP 重複指派——兩台同 IP 會衝突，症狀是間歇性斷線或 ARP 飄移。
